# DBil v0.1 — Plan 2 (Auth + HTTP server)

## Context

Plan 1 built the foundation: crypto layer, state store, `dbil init` creates an
admin user with a random password and writes a genesis audit entry. There is no
way to *use* that admin yet — no HTTP server, no login, no requireAuth.

Plan 2 closes that gap. It adds the user-facing surface (HTTP API and the
`dbil serve` subcommand) plus the security spine that every later plan will
mount on: session-backed auth, a single requireAuth middleware path for both
solo and team modes, and audit-logged auth events.

After Plan 2 an operator can:

1. `dbil serve` against the data dir from `dbil init`.
2. `POST /api/auth/login` with email/password from `initial-credentials.txt`.
3. Receive a session token and use it as `Authorization: Bearer <token>`.
4. Hit `GET /api/me` to confirm the session.
5. `POST /api/auth/logout` to revoke the session.
6. Every login (success and failure), logout, and rejected-token attempt is in
   the tamper-evident audit chain.

Still out of scope: Postgres connection management, SQL editor, observability,
frontend. Plans 3–9.

## Spec alignment

This plan implements section 9 of the spec (hybrid solo/team behaviour) and the
HTTP-server / auth middleware part of section 5. The static CI lint that enforces
`requireAuth()` on every handler (spec §6 risk table, "Hybrid solo/team auth
risk") is also in this plan.

## Tasks

### Phase A — Sessions storage

**Task 1 — Migration `0002_sessions`**
- Up: `sessions(id TEXT PK, user_id INT REFERENCES users(id) ON DELETE CASCADE,
  created_at INT NOT NULL, expires_at INT NOT NULL, revoked_at INT,
  user_agent TEXT, ip TEXT)`. `id` is `sha256(token)` hex (not the token itself).
- Indexes on `user_id` and `expires_at`.
- Down: drops the table.

**Task 2 — `SessionsRepo`**
- `Create(ctx, userID, ttl, ua, ip)` → `(token string, expiresAt time.Time, err)`.
- `Lookup(ctx, token)` → `(User, ExpiresAt, error)`. Rejects expired or revoked.
  Reads `users` via join for the returned User.
- `Revoke(ctx, token)` → updates `revoked_at = now`.
- `DeleteExpired(ctx, now)` → returns rowsAffected.
- Tests: roundtrip, expiry rejection, revocation rejection, unknown token rejection,
  cascade delete when user is removed.

### Phase B — Auth domain

**Task 3 — `internal/auth/auth.go`**
- `var ErrInvalidCredentials = errors.New("invalid credentials")` — surfaced
  identically for unknown-user *and* wrong-password to avoid user enumeration.
- `func VerifyPassword(ctx, db, email, password) (User, error)` — fetches user,
  computes Argon2id with stored salt, constant-time compare. Always runs the
  argon2id step even when the user is missing (timing equalisation).
- `func Login(ctx, deps, email, password, ua, ip) (token string, user User,
  expiresAt time.Time, err error)` — verifies, creates session (TTL: 7d solo,
  12h team — plan 2 only cares about solo, hard-codes 7d as `DefaultSessionTTL`),
  appends `auth.login` to audit. On failure appends `auth.login.failed`.
- `func Logout(ctx, deps, token) error` — revokes session, appends `auth.logout`.

**Task 4 — `internal/auth/middleware.go`**
- `type contextKey int; const userCtxKey contextKey = iota`.
- `func RequireAuth(deps Deps) func(http.Handler) http.Handler` — reads
  `Authorization: Bearer <token>`, calls `SessionsRepo.Lookup`, injects the
  User into the request context, sets `Request-User` response header for logs.
  On miss returns 401 with `{"error":"unauthorized"}` and writes
  `auth.token.rejected` audit event.
- `func UserFromContext(ctx) (User, bool)`.

### Phase C — HTTP server skeleton

**Task 5 — `internal/server/server.go`**
- `type Server struct{ http.Server; addr string }`.
- `func New(addr string, h http.Handler) *Server`.
- `Start(ctx)` blocks until the server exits.
- `Shutdown(ctx)` with a 5-second timeout.
- Uses chi `middleware.RequestID`, `middleware.Recoverer`, `middleware.Timeout(30s)`,
  and a custom slog-backed request logger.

**Task 6 — `internal/server/handlers/healthz.go`**
- `func Healthz(w, r)` returns 200 `{"status":"ok","version":<X>}`.
- No auth (allowlisted in lint-auth).

**Task 7 — `internal/server/handlers/auth.go`**
- `POST /api/auth/login` — body `{"email":...,"password":...}`. Returns
  `{"token":"<...>","expires_at":<unix>}`. Allowlisted (no auth).
- `POST /api/auth/logout` — under requireAuth. Reads Bearer token, revokes.
- `GET /api/me` — under requireAuth. Returns `{"id":...,"email":...,"role":...,
  "must_rotate":...}`.

**Task 8 — `internal/server/handlers/handlers.go`**
- `func Mount(r chi.Router, deps Deps)` — composes the router:
  ```
  r.Get("/healthz", Healthz)
  r.Post("/api/auth/login", LoginHandler(deps))
  r.Group(func(p chi.Router) {
      p.Use(auth.RequireAuth(deps))
      p.Post("/api/auth/logout", LogoutHandler(deps))
      p.Get("/api/me", MeHandler())
  })
  ```

### Phase D — CLI + e2e

**Task 9 — `cmd/dbil/serve_cmd.go`**
- `dbil serve` command, listens on `cfg.Port` (default 4242).
- Wires: master key load (loader chain identical to init), DB open + migrate,
  repositories, mount handlers, start server. Shuts down on SIGINT/SIGTERM
  with a 5-second grace period.

**Task 10 — `tests/e2e/serve_test.go`**
- Builds binary; `dbil init` into temp dir to seed admin; reads
  `initial-credentials.txt`; starts `dbil serve` as a background process on a
  random free port; waits for `/healthz`; POSTs login; asserts token; GETs
  `/api/me` with the token; LOGouts; second GET `/api/me` returns 401.

### Phase E — Static lint + finalize

**Task 11 — `scripts/lint-auth/main.go`**
- Parses `internal/server/handlers/*.go` AST; finds every function declaration
  matching `func ... Handler(deps Deps) http.HandlerFunc` (or the simpler
  `func Name(w http.ResponseWriter, r *http.Request)`); verifies the function
  is referenced either inside an `auth.RequireAuth(...)`-protected group in
  `handlers.go`'s `Mount` OR appears in an explicit unauthed allowlist
  (`Healthz`, `LoginHandler`). Otherwise exits non-zero with a clear message.
- Makefile target `lint-auth` runs it; CI adds it to the lint job.

**Task 12 — Final pass + tag**
- `make test cover lint-auth` — coverage ≥80% on `internal/auth`,
  `internal/server`, sessions; total still ≥80%.
- `go vet ./...` + (when installed) `govulncheck`, `golangci-lint`.
- Tag `v0.2.0-auth`.

## Risks acknowledged

- **No CSRF protection in Plan 2**: only Bearer-token APIs, no cookie auth yet,
  so CSRF is not a vector. When the SPA frontend lands (Plan 5) we will switch
  to SameSite cookies and add CSRF tokens.
- **No rate limit on login**: a quick `time.Sleep(200ms)` on failed VerifyPassword
  + the constant-time argon2id call together cap brute-force throughput at ~5 req/s
  per worker. Proper rate limit lands in Plan 4 / 5 together with the request
  middleware.
- **Solo-only TTL (7d)**: Plan 2 hardcodes 7d. Team-mode 12h and production 1h
  arrive in Plan 4 with tag-driven policies.
- **No /healthz auth allowlist enforcement at server level**: the lint script is
  the enforcement. If a future contributor adds an unauthed handler outside the
  allowlist, lint-auth fails.

## End-to-end verification

After Plan 2 ships:

```bash
DATA=$(mktemp -d) && ./bin/dbil init  # solve once for admin creds
./bin/dbil serve &
SERVE_PID=$!
sleep 1
curl -s http://localhost:4242/healthz                                                # 200 ok
TOKEN=$(curl -s -X POST http://localhost:4242/api/auth/login \
  -H 'Content-Type: application/json' \
  -d "{\"email\":\"admin@local\",\"password\":\"$(grep ^password "$DATA/initial-credentials.txt" | cut -d= -f2)\"}" \
  | jq -r .token)
curl -s -H "Authorization: Bearer $TOKEN" http://localhost:4242/api/me               # {id,email,role,must_rotate}
curl -s -X POST -H "Authorization: Bearer $TOKEN" http://localhost:4242/api/auth/logout
curl -s -o /dev/null -w '%{http_code}\n' -H "Authorization: Bearer $TOKEN" http://localhost:4242/api/me  # 401
kill $SERVE_PID
```

The e2e test in `tests/e2e/serve_test.go` encodes those steps mechanically.
