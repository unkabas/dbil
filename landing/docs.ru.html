полевой справочник

Быстрый старт.

dbil — это один контейнер ~20 МБ с React-интерфейсом внутри. Два пути установки: положить в существующий docker-compose.yml для авто-обнаружения, или запустить отдельно и добавлять соединения вручную. Выбирай тот, что подходит к месту, где уже живёт твой Postgres.

Быстрый старт

Эти два пути не «лучше или хуже» — они подходят к разным ситуациям. Вариант A для проектов, где уже есть compose, и dbil должен прочитать креды баз из лейблов контейнеров. Вариант B для всех остальных случаев: managed-облако, база на твоём ноутбуке, удалённая dev-VM.

Вариант A: авто-обнаружение через compose-лейблы

Добавь две вещи на свой сервис Postgres: набор dbil.* лейблов и общую сеть. Затем подключи dbil к этой сети и примонтируй сокет Docker только на чтение. dbil сканирует контейнеры на наличие своих лейблов и открывает соединения сам. Ввод кредов вручную не нужен.

Подними. На первом запуске пароль администратора пишется в файл /data/initial-credentials.txt внутри контейнера dbil, а также печатается в логи.

Открой http://localhost:4242, залогинься паролем администратора, и в сайдбаре должно уже висеть соединение App DB с тегом dev.

почему ro Сокет Docker монтируется с :ro, потому что dbil только читает лейблы контейнеров и резолвит значения env-переменных. Он никогда не запускает, не останавливает и не меняет контейнеры.

Вариант B: запустить отдельно, подключить в UI

Подходит, когда твой Postgres лежит там, где dbil не увидит лейблов: managed-облако, ноутбук, удалённая dev-VM, отдельный compose-проект. Запускаешь dbil сам по себе и добавляешь соединения через интерфейс рабочего места.

Открой http://localhost:4242, прочитай пароль администратора из логов (docker logs dbil), затем переходи к разделу «Добавление соединения» ниже.

Проверить образ

Каждый релиз-тег публикует multi-arch образ, подписанный cosign keyless OIDC. SPDX SBOM прикреплён к каждому релизу на GitHub.

Пароль администратора на первом запуске

При самом первом старте dbil генерирует пароль администратора и кладёт его в два места:

  • в логи dbil (одна строка с префиксом admin password:)
  • в файл /data/initial-credentials.txt внутри контейнера (на случай ротации логов)

Файл удаляется автоматически после первого успешного входа. Чтобы сбросить пароль, выйди и запусти docker exec dbil dbil reset-admin.

Добавление соединения

Если ты пошёл по Варианту A, соединения уже отображаются в сайдбаре. Процесс ниже — для Варианта B или для любой второй базы, которую ты добавляешь позже независимо от пути установки.

  1. Нажми Connections в левом сайдбаре.
  2. Нажми Add connection в шапке панели.
  3. Заполни host, port, database, user, password.
  4. Выбери тег: local, dev, staging или production. Тег — это контракт, по которому dbil решает, в чём отказывать; см. «Теги и production-безопасность».
  5. Нажми Test. dbil откроет соединение, выполнит SELECT 1 и прочитает pg_catalog. Зелёный значит готово.
  6. Save. Пароль сразу шифруется DEK, выведенным из мастер-ключа, и пишется в /data/dbil.db как шифротекст. На диске он не лежит в открытом виде ни секунды.

На соединениях с тегом production dbil дополнительно запросит отдельную парольную фразу, не выведенную из мастер-ключа. Эта фраза нужна при каждом открытии соединения в новой сессии.

фраза Потеряешь фразу — потеряешь доступ. Восстановления нет, потому что нет такого пути восстановления, который не был бы также путём атаки. Относись к production-фразе так, как ты относишься к ключу шифрования бэкапов базы.

Теги и production-безопасность

У каждого соединения есть тег. Тег определяет, какие запросы dbil отклоняет до того, как они уйдут из браузера, что рабочее место запрашивает у пользователя и что попадает в лог.

тег DDL DML DELETE / UPDATE без WHERE
local разрешено разрешено разрешено
dev разрешено X-Confirm отказ
staging отказ X-Confirm отказ
production отказ X-Confirm + фраза отказ

Отказ означает, что запрос возвращает HTTP/1.1 412 Precondition Required с сервера dbil, не доходя до Postgres. Браузер сразу показывает причину; отклонённый стейтмент записывается в цепочку аудита вместе с тегом.

X-Confirm значит, что DML-стейтмент удерживается, пока пользователь явно не подтвердит его через редактор (это ставит заголовок X-Confirm: yes на запрос). Подтверждение не сохраняется между стейтментами.

Таймауты стейтментов завязаны на тег: пять минут на local, тридцать секунд на dev, десять секунд на всё выше. Чтобы изменить их, отредактируй /data/dbil.db через dbil config edit.

Решение проблем

Контейнер dbil запустился, но соединений нет (Вариант A)

dbil обнаруживает только контейнеры в сети, указанной в DBIL_NETWORK. Если твой сервис Postgres в другой сети — добавь его в appnet или задай DBIL_NETWORK правильному имени. Проверка: docker network inspect appnet.

Потерял пароль администратора до первого входа

Пароль лежит в /data/initial-credentials.txt, пока ты не зайдёшь хотя бы раз. После этого: docker exec dbil dbil reset-admin.

Тест соединения отвечает «permission denied for table pg_class»

Postgres-юзеру нужны права на чтение pg_catalog. Дай ему pg_read_all_data или роль с явным SELECT на information_schema.

Граф схем пустой после подключения

dbil сканирует только схемы, перечисленные в поле visible schemas соединения (по умолчанию public). Открой соединение, добавь нужные схемы, нажми Refresh schema.
